# AGENTS.md

## Cursor Cloud specific instructions

This repository is a **native iOS application** (`EquilController`) and **cannot be
built, run, or tested on the Linux Cursor Cloud Agent VM**. Do not spend time
trying to install a toolchain to build it here — it is not possible.

Why it cannot run on this VM:

- **Build system is Xcode-only.** The project is generated by `xcodegen` from
  `project.yml` and built with `xcodebuild -project EquilController.xcodeproj`
  targeting `platform=iOS` with Apple code signing (`DEVELOPMENT_TEAM`,
  `-allowProvisioningUpdates`). `xcodegen`/`xcodebuild`/Xcode only exist on
  **macOS**. See `README.md` and `AGENT_HANDOFF.md` (§5 Build & install).
- **Apple-only frameworks.** Sources import `SwiftUI`, `CoreBluetooth`,
  `CryptoKit`, `Combine`, `Network`, and `os`. None of these are available in
  the open-source Swift toolchain for Linux, so even a partial compile of the
  app is not feasible.
- **Runtime requires physical hardware.** Running the app needs a physical
  **iOS device** (Xcode 26+), and exercising its core functionality requires a
  physical **Equil patch pump** over Bluetooth LE. There is no simulator path
  for the BLE pump protocol.

What this means for a Cloud Agent:

- There is **no update/setup script** and **no service** to run in this Linux
  environment. Lint/test/build/run all require a macOS + Xcode workstation.
- Do all real development, building, and device/BLE testing on **macOS with
  Xcode 26+**, following `README.md` and `AGENT_HANDOFF.md`.
- Code review, reading, and non-build edits are still possible here, but any
  change must be built and tested on macOS/hardware before it can be validated.
- `AGENT_HANDOFF.md` is the authoritative technical handoff (architecture, the
  critical "RUN mode" gotcha for SET commands, and per-command status). It is
  written in Hungarian.
